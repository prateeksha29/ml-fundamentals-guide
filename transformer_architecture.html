1. The Big Picture

📄
Attention Is All You Need

The original Transformer was designed for machine translation (English-to-German, English-to-French), but its architecture turned out to be far more general. It's an encoder-decoder model built entirely from attention and feedforward layers — no recurrence, no convolutions.

2. The Input Pipeline: From Words to Vectors

2.1 Tokenization

The original Transformer used byte-pair encoding (BPE), a subword tokenization algorithm (Sennrich et al., 2015). BPE starts with individual characters and iteratively merges the most frequent adjacent pairs. This produces a vocabulary of subword units that balances between character-level (too fine-grained, very long sequences) and word-level (too large vocabulary, can't handle unseen words).

For example: "unhappiness" might tokenize as ["un", "happiness"] or ["un", "happ", "iness"]. This allows the model to handle rare or novel words by composing known subword pieces.

2.2 Token Embeddings

Each token maps to a learned \(d_{\text{model}}\)-dimensional vector via an embedding matrix \(E \in \mathbb{R}^{V \times d_{\text{model}}}\). An important detail: the same embedding matrix is shared between encoder input, decoder input, and the decoder's output projection (weight tying, Press & Wolf 2016), with weights multiplied by \(\sqrt{d_{\text{model}}}\).

💡 Why multiply by √d_model?

Embedding vectors have values \(\sim 1/\sqrt{d_{\text{model}}}\) (from initialization), while sinusoidal positional encodings range in \([-1, 1]\). Without scaling, the positional signal would dominate. Multiplying embeddings by \(\sqrt{d_{\text{model}}}\) brings both to the same scale.

2.3 Positional Encoding

Self-attention is permutation-equivariant — "Dog bites man" and "Man bites dog" produce identical attention without position info. The Transformer adds sinusoidal positional encodings:

Sinusoidal Positional Encoding
$$PE(\text{pos}, 2i) = \sin\!\left(\frac{\text{pos}}{10000^{2i/d_{\text{model}}}}\right)$$ $$PE(\text{pos}, 2i+1) = \cos\!\left(\frac{\text{pos}}{10000^{2i/d_{\text{model}}}}\right)$$ $$\text{input} = \text{Embedding}(x) \cdot \sqrt{d_{\text{model}}} + PE(\text{pos})$$

Each dimension oscillates at a different frequency — dimension 0 has the highest frequency (period \(2\pi\)), higher dimensions have progressively lower frequencies (up to \(10000 \cdot 2\pi\)).

⚡ Why sinusoidal? The key property

For any fixed offset \(k\), \(PE(\text{pos} + k)\) is a linear transformation of \(PE(\text{pos})\): specifically, \(PE(\text{pos}+k) = M_k \cdot PE(\text{pos})\) for a rotation matrix \(M_k\) that depends only on \(k\), not \(\text{pos}\). This means the model can learn to attend to relative positions ("3 tokens back") regardless of absolute position — enabling extrapolation to longer sequences than seen during training.

The paper also tested learned positional embeddings and found nearly identical results. They chose sinusoidal for the extrapolation advantage.

3. The Encoder: Building Rich Representations

The encoder is a stack of N = 6 identical layers. Each layer has two sub-layers, each wrapped with a residual connection and LayerNorm:

Figure 2: Single encoder layer. Each sub-layer (attention and FFN) is wrapped with a residual connection (dashed line + ⊕) and LayerNorm. The output dimension is always d_model = 512.

3.1 Multi-Head Self-Attention

In the encoder, self-attention allows every token to attend to every other token — fully bidirectional, no masking.

Scaled Dot-Product Attention
$$Q = XW_Q, \quad K = XW_K, \quad V = XW_V$$ $$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V$$

Dimensionality at each step (for one head, \(d_k = 64\)):

Figure 3: Data flow through one attention head. X (n × 512) is projected to Q, K, V (n × 64 each). QK⊤ produces an n × n score matrix, softmax normalizes to weights, and multiplication by V gives the n × 64 output.

Q, K, V Intuition — information retrieval:

  • Q (Query): "What am I looking for?" — what information does this position need?
  • K (Key): "What do I contain?" — what does this position advertise?
  • V (Value): "What do I provide if matched?" — the content that gets passed along.

The dot product \(Q \cdot K^\top\) measures compatibility: high score → high attention weight → that position's Value gets incorporated strongly. Having separate learned projections for Q, K, V (rather than using X directly) gives the model far more expressiveness — the query can encode "what I need" differently from the key's "what I have."

Multi-head attention runs \(h = 8\) parallel attention heads, each with \(d_k = 64\):

Multi-Head Attention
$$\text{head}_i = \text{Attention}(XW_Q^i,\; XW_K^i,\; XW_V^i) \quad \in \mathbb{R}^{n \times 64}$$ $$\text{MultiHead}(X) = \text{Concat}(\text{head}_1, \ldots, \text{head}_8) \cdot W_O \quad \in \mathbb{R}^{n \times 512}$$

The original paper showed that individual attention heads learn distinct patterns — some attend to adjacent tokens (local syntax), others capture long-range dependencies (coreference, subject-verb agreement). Multi-head attention provides an inductive bias toward learning diverse, complementary patterns.

3.2 Residual Connections & Layer Normalization

Each sub-layer is wrapped: output = LayerNorm(x + SubLayer(x))

Residual Connection — Why It Works
$$\frac{\partial(x + f(x))}{\partial x} = I + \frac{\partial f(x)}{\partial x}$$

The identity \(I\) ensures gradients always flow, even if \(f'(x)\) is small. Without residuals, the Transformer's 24+ sub-layers would suffer from vanishing gradients. Borrowed from ResNets (He et al., 2016).

Layer Normalization (Ba et al., 2016)
$$\text{LayerNorm}(x) = \gamma \cdot \frac{x - \mu}{\sigma + \epsilon} + \beta$$

where μ, σ are computed across the d_model dimensions for each position (not across the batch)

💡 Why LayerNorm, not BatchNorm?

BatchNorm computes stats across the batch — requires large batches, behaves differently at train vs. inference, and breaks with variable-length sequences. LayerNorm computes stats across features per-sample — batch-size independent, identical at train/inference, and works for autoregressive generation where batch size is effectively 1.

Pre-norm vs. Post-norm: The original Transformer uses post-norm — normalize after the residual add. Most modern models (GPT-2/3, Llama) use pre-norm — normalize before the sub-layer: \(x + \text{SubLayer}(\text{LayerNorm}(x))\). Pre-norm is more stable to train (gradient flow through the residual stream is cleaner), while post-norm may achieve slightly better final performance when training succeeds.

3.3 Position-wise Feed-Forward Network

FFN — Two Linear Layers with ReLU
$$\text{FFN}(x) = \max(0,\; xW_1 + b_1) \cdot W_2 + b_2$$

W₁ ∈ ℝ^(512 × 2048), W₂ ∈ ℝ^(2048 × 512) — a 4× expansion then contraction

Figure 4: FFN expand-and-contract. The 4× expansion provides non-linear transformation capacity that attention alone lacks. The FFN dominates the parameter count.
⚠️ Interview question: Why can't you just stack attention layers without FFN?

Attention computes weighted averages of value vectors — a convex combination (softmax weights ≥ 0, sum to 1). This is linear in V. The FFN provides: (1) Non-linearity through ReLU/GELU, (2) Per-position independent processing (attention mixes across positions, FFN processes each separately), (3) Knowledge storage — Geva et al. (2021) showed FFN weights act as key-value memories for factual associations.

3.4 Dropout

Applied in three places (rate = 0.1 in base model): (1) after embedding + positional encoding, (2) on attention weights after softmax (before multiplying by V), (3) on each sub-layer output before the residual add. Active during training, disabled at inference — the full attention pattern is used at test time.

4. The Decoder: Generating Output Autoregressively

The decoder is also N = 6 identical layers, but each has three sub-layers:

Figure 5: Single decoder layer. Three sub-layers: (1) masked self-attention (can't see future), (2) cross-attention to encoder output (Q from decoder, K/V from encoder), (3) FFN. Each wrapped with residual + LayerNorm.

4.1 Masked Self-Attention

During training, the entire target sequence is fed at once (teacher forcing), but a causal mask prevents position \(i\) from attending to positions \(j > i\):

Causal Mask
$$\text{mask}(i,j) = \begin{cases} 0 & \text{if } j \leq i \text{ (allowed)} \\ -\infty & \text{if } j > i \text{ (blocked)} \end{cases}$$ $$\text{attention} = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}} + \text{mask}\right) V$$
Figure 6: Causal mask for decoder self-attention. Green = allowed (✓), red = blocked (-∞). Position 2 can attend to positions 1-2 but not 3-4. This prevents "peeking" at future tokens during training.

4.2 Cross-Attention

The bridge between encoder and decoder. Q comes from the decoder, K and V come from the encoder output:

Cross-Attention
$$Q = \text{decoder\_output} \cdot W_Q^{\text{cross}}$$ $$K = \text{encoder\_output} \cdot W_K^{\text{cross}}, \quad V = \text{encoder\_output} \cdot W_V^{\text{cross}}$$

This allows each decoder position to attend to all encoder positions — exactly like Bahdanau attention but using the Transformer's scaled dot-product formulation with multi-head attention. The attention weights form a soft alignment between source and target sentences.

4.3 Teacher Forcing & Inference

Training (teacher forcing): The full target sequence is fed at once with causal masking, allowing parallel processing. Without this, we'd decode one token at a time during training — losing all parallelism.

Inference (autoregressive): Generate one token at a time. Start with [BOS], run encoder once, then iteratively: decode → sample next token → feed back as input. O(n) sequential steps for n output tokens.

💡 Exposure bias

During training, the decoder conditions on correct previous tokens (teacher forcing). During inference, it conditions on its own (possibly incorrect) predictions. This mismatch is called exposure bias. Techniques like scheduled sampling and beam search partially address it.

5. Training: The Noam Schedule and Label Smoothing

5.1 Learning Rate Schedule

The "Noam schedule" (named after co-author Noam Shazeer) combines linear warmup with inverse square root decay:

Noam Learning Rate Schedule
$$lr = d_{\text{model}}^{-0.5} \cdot \min\!\left(\text{step}^{-0.5},\; \text{step} \cdot \text{warmup}^{-1.5}\right)$$

warmup_steps = 4000 · Linear warmup for first 4000 steps, then 1/√step decay

Figure 7: The Noam learning rate schedule. Linear warmup prevents destructive updates when attention patterns are random, then inverse square root decay allows fine-grained convergence.
⚡ Why warmup is essential

At initialization, attention weights are essentially random — each token attends uniformly to all others. Gradients are chaotic. A large learning rate would cause destructive parameter updates. Warmup starts tiny, letting the model establish reasonable attention patterns before ramping up. The paper used Adam with β₁ = 0.9, β₂ = 0.98 (unusually high β₂ for faster second-moment adaptation).

5.2 Label Smoothing

Instead of hard one-hot targets, the paper uses label smoothing with \(\epsilon = 0.1\):

Label Smoothing
$$\text{target} = (1 - \epsilon) \cdot \text{one\_hot} + \frac{\epsilon}{V}$$

Correct class gets 0.9, remaining 0.1 spread across all V vocabulary tokens

This prevents overconfidence (model trying to push logits to ±∞, causing softmax saturation and vanishing gradients), improves calibration, and acts as regularization. The paper reported that label smoothing hurt perplexity but improved BLEU — better translations at the cost of less precise probability estimates.

9. Interview Questions

Q1 — Walk-Through
Walk me through the Transformer architecture end-to-end.
Start: encoder-decoder, N=6 layers each. Input pipeline: tokenization (BPE) → embedding × √d_model → add positional encoding → dropout. Encoder layer: multi-head self-attention (8 heads, no masking) → add & norm → FFN (512→2048→512 with ReLU) → add & norm. Decoder layer: masked self-attention (causal mask) → add & norm → cross-attention (Q from decoder, K/V from encoder) → add & norm → FFN → add & norm. Output: linear projection to vocab size → softmax. Training: Adam with Noam schedule (4000 warmup), label smoothing ε=0.1. Practice giving this as a fluent 3-5 minute explanation.
Q2 — Detail
Why does the Transformer multiply the embedding by √d_model?
Embeddings are initialized with values ~\(1/\sqrt{d_{\text{model}}}\), while sinusoidal positional encodings have values in [-1, 1]. Without scaling, position signal dominates semantic signal. Multiplying by \(\sqrt{d_{\text{model}}}\) brings both to the same scale.
Q3 — Critical
What's the role of the FFN? Why can't you just use attention?
Attention computes weighted averages of value vectors — linear in V (convex combination since softmax weights ≥ 0, sum to 1). The FFN provides: (1) non-linearity (ReLU/GELU), (2) per-position independent processing (attention mixes across positions, FFN processes each separately), (3) knowledge storage — Geva et al. (2021) showed FFN weights act as key-value memories. The FFN has ~2/3 of each layer's parameters.
Q4 — Detail
Where is dropout applied, and what changes at inference?
Three places: (1) after embedding + PE sum, (2) on attention weights after softmax (before ×V), (3) on each sub-layer output before residual add. Rate = 0.1. During training, randomly zeros activations. At inference, completely disabled — full attention pattern used. This distinction is important because the model uses undropped attention at test time.
Q5 — Decoder
How does masking work in the decoder's self-attention?
A causal mask is added to scores before softmax: entries (i, j) where j > i are set to \(-\infty\). After softmax, these become 0 — each position can only attend to itself and earlier positions. During training, the full target is fed at once (teacher forcing) with this mask simulating autoregressive behavior. At inference, masking is implicit since only generated tokens exist.
Q6 — Comparison
What's the difference between self-attention and cross-attention in the decoder?
Self-attention: Q, K, V all from the decoder itself, with causal masking. Cross-attention: Q from decoder, K and V from encoder output. Cross-attention lets the decoder "read" the source sentence — functionally equivalent to Bahdanau attention but using the Transformer's scaled dot-product formulation with multi-head attention.
Q7 — Normalization
Why LayerNorm instead of BatchNorm?
BatchNorm: stats across batch → requires large batches, different behavior at train vs. inference, breaks with variable-length sequences. LayerNorm: stats across features per-sample → batch-size independent, identical train/inference behavior, works for autoregressive generation (effective batch size = 1).
Q8 — Training
Why do we need residual connections?
Gradient highway: \(\partial(x + f(x))/\partial x = 1 + f'(x)\) — the "1" ensures gradients always flow. Without residuals, the Transformer's 24+ sub-layers would cause vanishing gradients and training failure. Also makes optimization easier — the network learns the residual \(f(x)\) (deviation from identity), not the full transformation.
Q9 — Training
Explain the Noam learning rate schedule. Why is warmup necessary?
Formula: \(lr = d_{\text{model}}^{-0.5} \cdot \min(\text{step}^{-0.5},\, \text{step} \cdot \text{warmup}^{-1.5})\). Linear warmup for 4000 steps, then \(1/\sqrt{\text{step}}\) decay. Warmup is needed because at initialization, attention weights are essentially random (uniform). Gradients are noisy. Large lr → destructive updates. Warmup lets reasonable attention patterns form first.
Q10 — Depth
Why 8 attention heads? What happens with 1 or 32?
8 heads × d_k=64 was empirically optimal for d_model=512. With 1 head (d_k=512): single attention pattern, less diversity, worse performance. With 32 heads (d_k=16): each head has too little capacity for meaningful patterns — also worse. The paper's ablation confirmed both extremes hurt. Sweet spot: heads large enough to be useful individually, numerous enough for diversity.
Q11 — Teacher Forcing
Why teacher forcing, and what's the mismatch at inference?
Teacher forcing feeds ground-truth tokens as decoder input during training, allowing parallel processing (full sequence + mask). Without it: O(n) forward passes per example, losing all parallelism. The mismatch: training conditions on correct tokens; inference conditions on the model's own (possibly wrong) predictions. This is called exposure bias. Beam search and scheduled sampling partially address it.
Q12 — Efficiency
The base model trained in 12 hours on 8 GPUs. What made this possible vs. RNNs taking weeks?
Two things: (1) Full parallelism — entire source and target processed simultaneously (with masking), vs. RNNs requiring O(n) sequential steps. Massive GPU utilization. (2) Simpler compute graph — matrix multiplications and softmax are highly optimized on GPUs; RNN sequential dependencies create pipeline stalls. More data per wall-clock time → faster convergence.
Q13 — Application
If you were building a recommendation system for Prime Video and needed to encode user watch histories, how would the Transformer encoder help?
A user's watch history is a sequence of items — suited for a Transformer encoder. Self-attention captures co-viewing patterns: a thriller followed by a documentary signals different intent than two comedies. The FFN encodes item-level features (genre, length, recency). Positional encoding captures temporal ordering. Unlike RNN approaches, the Transformer processes the entire history in parallel with O(1) path length for long-range patterns (e.g., seasonal rewatching). This is the architecture behind SASRec and BERT4Rec.
