01Why Activation Functions Matter

Without activation functions, a neural network is just stacked linear transformations — which collapse into a single linear transformation no matter how many layers you add. A two-layer network W₂(W₁x + b₁) + b₂ simplifies to Wx + b. Activation functions break this linearity and let networks learn complex, non-linear decision boundaries.

The choice of activation function affects training dynamics (gradient flow, convergence speed), the kinds of functions a network can represent, and computational cost. In interviews, you need to know the formulas, the derivatives, the failure modes, and why each one was introduced — meaning what problem with the previous generation it fixed.

02Sigmoid

classic binary classification output gating mechanism
σ(x) = 1 / (1 + e⁻ˣ)

Sigmoid squashes any real number into (0, 1), which made it the go-to activation in the early days of neural networks. It has a natural probabilistic interpretation — the output can be read as a probability — which is why it's still used in binary classification output layers and as gating functions inside LSTMs and GRUs.

Derivative

σ'(x) = σ(x) · (1 − σ(x))

The derivative peaks at 0.25 (when x=0) and approaches 0 for large |x|. This is the root of the vanishing gradient problem: during backpropagation through many layers, gradients get multiplied by numbers ≤ 0.25 repeatedly, shrinking to near zero.

Sigmoid function (solid) and its derivative (dashed). The derivative never exceeds 0.25.

Implementation

# NumPy — numerically stable version
def sigmoid(x):
    return np.where(x >= 0,
                    1 / (1 + np.exp(-x)),
                    np.exp(x) / (1 + np.exp(x)))

def sigmoid_derivative(x):
    s = sigmoid(x)
    return s * (1 - s)
Numerical stability

The naive 1/(1+exp(-x)) overflows for large negative x. The piecewise version above handles both sides. This is a common follow-up in coding interviews.

Where it's used today

Binary classification output layers, gating in LSTMs/GRUs, attention mechanisms (as part of sigmoid attention variants). Rarely used as a hidden layer activation anymore because of vanishing gradients and non-zero-centered output.

03Tanh

classic zero-centered RNN/LSTM staple
tanh(x) = (eˣ − e⁻ˣ) / (eˣ + e⁻ˣ) = 2σ(2x) − 1

Tanh is sigmoid rescaled to (−1, 1). The key improvement: it's zero-centered, meaning its outputs average around 0 rather than 0.5. This matters because non-zero-centered activations cause zig-zagging updates during gradient descent — all gradients for a layer's weights have the same sign, forcing updates in only one direction at a time.

Derivative

tanh'(x) = 1 − tanh²(x)

Peaks at 1.0 (when x=0), which is 4× stronger than sigmoid's peak. Still saturates for large |x|, so it doesn't fully solve vanishing gradients, but it's meaningfully better than sigmoid for hidden layers.

Tanh (solid) and its derivative (dashed). Derivative peak is 4× that of sigmoid.

Implementation

def tanh(x):
    return np.tanh(x)  # or: (np.exp(x) - np.exp(-x)) / (np.exp(x) + np.exp(-x))

def tanh_derivative(x):
    return 1 - np.tanh(x) ** 2

Where it's used today

Cell state updates in LSTMs, hidden state computation in GRUs and vanilla RNNs, and as a final activation in some generative models (e.g., the output layer of GANs to produce values in [−1, 1]). Also appears inside the original transformer's positional encoding isn't an activation per se, but tanh shows up in gating mechanisms throughout sequence models.

04ReLU

modern standard CNNs default hidden activation
ReLU(x) = max(0, x)

ReLU was the activation that made deep networks actually trainable. Published by Nair & Hinton (2010) and popularized by AlexNet (2012), it solved the vanishing gradient problem in a brutally simple way: for positive inputs, the gradient is always 1. No saturation, no squashing, no exponentials. Just a hinge.

It also introduces sparsity — any negative input produces exactly 0, meaning a subset of neurons are "off" for any given input. This acts as a form of implicit regularization and makes representations more interpretable.

Derivative

ReLU'(x) = 1 if x > 0, else 0   (undefined at x=0, conventionally set to 0)
ReLU and its derivative. Negative inputs produce zero gradient — the "dying ReLU" region.

Implementation

def relu(x):
    return np.maximum(0, x)

def relu_derivative(x):
    return (x > 0).astype(float)
The dying ReLU problem

If a neuron's weights get pushed so that its input is always negative, it outputs 0 for every sample and receives 0 gradient — it's permanently dead. This can happen with a large learning rate or bad initialization. A network can lose 20-40% of its neurons this way. Leaky ReLU and PReLU were invented specifically to fix this.

Where it's used today

Still the default activation for CNNs and most feedforward networks. ResNets, VGG, all the classic vision architectures use ReLU. For transformers and LLMs, GELU has largely replaced it (more on that below), but ReLU remains the starting point for any new architecture and the baseline everyone compares against.

05Leaky ReLU & PReLU

modern fixes dying ReLU GANs
LeakyReLU(x) = x if x > 0, else αx   (typically α = 0.01)

PReLU(x) = x if x > 0, else αx   (α is a learnable parameter)

Leaky ReLU applies a small slope α to negative inputs instead of hard zero. This means every neuron always has a non-zero gradient and can never fully die. PReLU takes this further by making α a learned parameter — the network itself decides the optimal negative slope, which can vary per channel.

He et al. (2015) showed PReLU improved ImageNet classification accuracy and that the learned α values were typically small (0.1–0.3 range), suggesting the network preferred a gentle leak over either hard zero or large negative pass-through.

Derivative

LeakyReLU'(x) = 1 if x > 0, else α
Leaky ReLU vs ReLU. The small negative slope keeps gradients flowing everywhere.

Implementation

def leaky_relu(x, alpha=0.01):
    return np.where(x > 0, x, alpha * x)

def leaky_relu_derivative(x, alpha=0.01):
    return np.where(x > 0, 1.0, alpha)

# PReLU: alpha is a parameter updated via backprop
class PReLU:
    def __init__(self, alpha=0.25):
        self.alpha = alpha  # learnable

    def forward(self, x):
        self.x = x
        return np.where(x > 0, x, self.alpha * x)

    def backward(self, grad_output):
        grad_x = np.where(self.x > 0, grad_output, self.alpha * grad_output)
        grad_alpha = np.sum(np.where(self.x > 0, 0, self.x * grad_output))
        return grad_x, grad_alpha

Where it's used today

Leaky ReLU is very popular in GANs (DCGAN architecture uses it for the discriminator). PReLU was used in the He et al. ResNet variants. In practice, Leaky ReLU is a safe drop-in replacement for ReLU when you suspect dying neurons — it almost never hurts and sometimes helps.

06Softmax & Log-Softmax

ubiquitous multi-class output attention mechanism
Softmax(xᵢ) = eˣⁱ / Σⱼ eˣʲ

LogSoftmax(xᵢ) = xᵢ − log(Σⱼ eˣʲ)

Softmax converts a vector of raw logits into a probability distribution — outputs sum to 1, each is in (0, 1). It's the standard final layer for multi-class classification. The "max" in the name comes from the fact that it approximates argmax: the largest logit gets disproportionately more probability as the differences grow.

Log-Softmax is the log of softmax, computed more stably in a single pass. It's numerically preferred when you're going to take the log anyway (e.g., for NLLLoss), since log(softmax(x)) computed naively can underflow.

The numerical stability trick

Raw softmax overflows for large logits. The standard fix: subtract the max value first. Since softmax(x) = softmax(x - c) for any constant c, we use c = max(x) to keep exponentials manageable.

Implementation

def softmax(x):
    # x shape: (batch, classes) or (classes,)
    shifted = x - np.max(x, axis=-1, keepdims=True)  # stability trick
    exp_x = np.exp(shifted)
    return exp_x / np.sum(exp_x, axis=-1, keepdims=True)

def log_softmax(x):
    shifted = x - np.max(x, axis=-1, keepdims=True)
    return shifted - np.log(np.sum(np.exp(shifted), axis=-1, keepdims=True))
Interview favorite

"Implement softmax from scratch" is one of the most common ML coding questions. The follow-ups are always: (1) how do you handle numerical overflow? (2) can you do it for a batch of inputs? (3) what's the Jacobian of softmax? The Jacobian answer: ∂softmax(xᵢ)/∂xⱼ = softmax(xᵢ)(δᵢⱼ − softmax(xⱼ)), where δᵢⱼ is the Kronecker delta.

Where it's used today

Final layer of virtually every classification model. Inside every transformer's attention mechanism (the attention weights are a softmax over query-key dot products). Temperature-scaled softmax is used in knowledge distillation and LLM sampling.

07GELU

modern — 2016 transformers BERT, GPT, ViT
GELU(x) = x · Φ(x)

where Φ(x) is the CDF of the standard normal distribution

Approximation: GELU(x) ≈ 0.5x(1 + tanh(√(2/π)(x + 0.044715x³)))

GELU (Hendrycks & Gimpel, 2016) is the activation that powers modern transformers. The idea: instead of a hard 0/1 gate like ReLU, weight each input by the probability that it's "above average" — specifically, by the CDF of a standard normal. Inputs that are clearly positive pass through almost fully; inputs that are clearly negative are nearly zeroed; and inputs near zero get a smooth, stochastic-feeling blend.

Unlike ReLU's hard kink at 0, GELU is smooth everywhere. This matters for optimization — smoother loss landscapes generally mean more stable training. The non-monotonic behavior (GELU has a small negative "bump" near x ≈ -0.5) is sometimes cited as helping with regularization, though the evidence is mixed.

GELU vs ReLU. Note the smooth transition and the small negative region near x = −0.5.

Implementation

# Exact
from scipy.stats import norm
def gelu_exact(x):
    return x * norm.cdf(x)

# Tanh approximation (what PyTorch uses)
def gelu_approx(x):
    return 0.5 * x * (1 + np.tanh(np.sqrt(2 / np.pi) * (x + 0.044715 * x**3)))

# Sigmoid approximation (faster, less accurate)
def gelu_sigmoid(x):
    return x * sigmoid(1.702 * x)
Why interviewers love this

"Why does BERT use GELU instead of ReLU?" tests whether you understand the tradeoffs: smoothness for optimization, probabilistic interpretation, and that the specific choice was somewhat empirical — GELU consistently gave small accuracy improvements in the transformer setting. Also know that the tanh approximation exists because the exact CDF is expensive to compute.

Where it's used today

BERT, GPT-2, GPT-3, GPT-4, ViT (Vision Transformer), RoBERTa, and most modern transformer architectures. If you see a transformer from 2018 onward, it's probably using GELU. This is the single most important "modern" activation function to know for LLM-focused interviews.

08Swish / SiLU

modern — 2017 EfficientNet, LLaMA NAS-discovered
Swish(x) = x · σ(βx)

SiLU is Swish with β = 1:   SiLU(x) = x · σ(x)

Swish was found via automated search (Neural Architecture Search) by Ramachandran et al. at Google Brain (2017). The search explored combinations of unary and binary operations and consistently found that x · σ(x) outperformed ReLU on deep networks. SiLU (Sigmoid Linear Unit) is the name for the β=1 special case, which is what's used in practice.

Like GELU, Swish is smooth and non-monotonic. The two functions are actually very similar in shape — they differ mainly in the size of the negative bump and the transition sharpness. The practical difference in performance is small; the choice between GELU and Swish often comes down to framework conventions.

Swish, GELU, and ReLU. Swish and GELU are nearly identical — both are smooth, non-monotonic alternatives to ReLU.

Implementation

def swish(x, beta=1.0):
    return x * sigmoid(beta * x)

def silu(x):
    # SiLU = Swish with beta=1
    return x * sigmoid(x)

def silu_derivative(x):
    s = sigmoid(x)
    return s + x * s * (1 - s)  # = s(1 + x(1-s))

Where it's used today

EfficientNet (Google, 2019), LLaMA and LLaMA 2 (Meta, 2023), Mamba (state space models), and many recent architectures. LLaMA specifically uses SiLU in its SwiGLU feed-forward layers, which combine Swish/SiLU with gated linear units. If you're interviewing at Meta or for any role involving open-source LLMs, know SiLU and SwiGLU.

The timeline of dominance

Pre-2010: Sigmoid and tanh were the only options. Deep networks couldn't train well.

2010–2015: ReLU era. AlexNet proved it worked, deep learning exploded. Leaky ReLU and PReLU appeared as patches.

2016–present: Smooth activations. GELU arrived with BERT (2018), Swish from NAS (2017). SwiGLU in LLaMA (2023) is the current frontier for large language models.

10Interview Questions

These range from basics to the follow-ups that separate strong candidates. I've included the key points for each answer — not scripted responses, because you want to explain these in your own words.

Why do we need non-linear activation functions?
Without them, stacking layers is pointless — the composition of linear functions is linear. A 100-layer network with no activations has the same representational power as a single layer. Non-linearity lets the network approximate arbitrary functions (universal approximation theorem). Follow-up trap: "Is ReLU technically non-linear?" Yes — it's piecewise linear, which is non-linear. The kink at 0 breaks linearity.
Why did ReLU replace sigmoid/tanh for hidden layers?
Three reasons: (1) no saturation for positive inputs, so gradients don't vanish during backprop; (2) extremely cheap to compute (just a threshold); (3) induces sparsity, which can help generalization. The deeper the network, the more sigmoid/tanh suffer from gradient attenuation — gradients shrink exponentially with depth since each layer multiplies by a factor ≤ 0.25 (sigmoid) or ≤ 1.0 (tanh).
Explain the dying ReLU problem and how to fix it.
If a neuron's pre-activation is consistently negative (due to a large negative bias or bad weight update), ReLU outputs exactly 0 and the gradient is exactly 0, so the weights never update — the neuron is permanently dead. Fixes: Leaky ReLU (small α for negative inputs), PReLU (learned α), ELU (exponential for negatives), or better initialization (He initialization, designed specifically for ReLU). You can also use a smaller learning rate to avoid the large updates that push neurons into the dead zone.
Compare sigmoid vs tanh. When would you use each?
Tanh is sigmoid scaled to (−1, 1) — literally tanh(x) = 2σ(2x) − 1. Tanh is zero-centered, which means layer outputs don't have a systematic positive bias, leading to more balanced gradient updates. Use sigmoid for: binary classification outputs (probability interpretation), gating (LSTM forget/input gates). Use tanh for: hidden state computations in RNNs, any place you want zero-centered outputs. Neither is great as a hidden layer activation in deep feedforward nets anymore — use ReLU or GELU.
Implement softmax from scratch. How do you make it numerically stable?
Subtract max(x) before exponentiating. Since softmax(x) = softmax(x - c) for any constant c, this doesn't change the result but prevents overflow. The follow-up: "What's the gradient of softmax?" The Jacobian is diag(s) - s·sᵀ, where s is the softmax output vector. In practice, you don't compute this explicitly — you use cross-entropy loss which simplifies the gradient to softmax(x) - y_onehot.
Why do transformers use GELU instead of ReLU?
GELU is smooth everywhere (no kink at 0), which helps with optimization on the complex loss landscapes of transformer training. It can be interpreted as stochastic regularization — each input is weighted by the probability that it exceeds random inputs from a standard normal. Empirically, GELU consistently outperformed ReLU and ELU on transformer benchmarks, though the margins are small. The real answer is partly empirical: the BERT authors tried it, it worked better, and it became the convention. Know the tanh approximation formula — it comes up in implementation questions.
What is SwiGLU and why does LLaMA use it?
SwiGLU (Shazeer, 2020) combines Swish activation with Gated Linear Units. In a standard transformer FFN, you have: FFN(x) = W₂ · GELU(W₁x). In SwiGLU, it becomes: FFN(x) = W₂ · (Swish(W₁x) ⊙ W₃x), where ⊙ is element-wise multiplication and W₃ is an additional projection. The gating mechanism lets the network multiplicatively control information flow, which turns out to give better perplexity per parameter. LLaMA, PaLM, and several other recent LLMs use it because it consistently improves performance at the cost of one extra projection matrix.
If you're designing a new network from scratch, how do you choose the activation function?
Start with the conventions for your architecture type. Transformers: GELU or SwiGLU for hidden layers, softmax for attention and classification output. CNNs: ReLU, or Leaky ReLU if you're worried about dead neurons. RNNs/LSTMs: tanh for cell states, sigmoid for gates. Binary output: sigmoid. Multi-class output: softmax. If you're doing architecture search or have compute to ablate, try Swish/SiLU as an alternative to GELU. The honest answer is that the differences between modern smooth activations are small — the choice of learning rate, normalization, and architecture matter more.
What's the relationship between softmax temperature and model behavior?
softmax(x/T) where T is temperature. T → 0 makes it approach hard argmax (peaky, confident). T → ∞ makes it uniform (all classes equally likely). T = 1 is standard. Low temperature is used for greedy decoding in LLMs. High temperature gives more diverse/creative outputs. Temperature scaling is also used in knowledge distillation (Hinton et al., 2015) — training a student model on soft targets from a teacher model with T > 1, which transfers "dark knowledge" about class relationships.
ReLU is not differentiable at x = 0. Is that a problem?
In theory, yes — the function isn't differentiable at exactly 0. In practice, no: (1) the probability of hitting exactly 0.0 in floating point is negligible; (2) we define the subgradient as 0 at that point by convention; (3) networks train perfectly fine with this. The interviewer is testing whether you understand the gap between mathematical theory and practical deep learning. Same idea applies to Leaky ReLU at x = 0.
